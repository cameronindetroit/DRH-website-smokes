## Description

This PR makes the smoke test selection data-driven and more robust. It replaces hard-coded state picks with a maintained `states.json` data file, adds a seeded RNG helper for reproducible smoke picks, and hardens page selectors used during state search and navigation to avoid ambiguous matches.

Key motivations:
- Make smoke tests representative of different states/regions without changing test code.
- Allow reproducible runs via `SMK_SEED` while keeping true-random as the default.
- Fix flaky failures caused by ambiguous autocomplete/listing link text (e.g., "Kansas" vs "Arkansas").

## Changes

- Added: `src/testdata/states.json` — canonical state + city data used by smoke tests (generated by `scripts/scrape_drhorton_states.mjs`).
- Added: `src/utils/RandomUtil.ts` — seeded `mulberry32` RNG and helper to pick distinct indexes; supports `SMK_SEED`.
- Added: `src/utils/RotationUtil.ts` — deterministic rotation helper (keeps previous rotation behavior available).
- Updated: `src/tests/smoke/home/homeSmoke.spec.ts` — now loads `states.json`, selects states via `RandomUtil` (or `RotationUtil` if configured), and attaches selection metadata to test info.
- Updated: `src/pages/HomePage.ts` and `src/pages/StateLandingPage.ts` — hardened `selectStateFromSearch` / `searchAndOpenState` using an exact-match algorithm for listbox options and safe link fallbacks.
- Added: `scripts/scrape_drhorton_states.mjs` and `scripts/inspect_state_option.mjs` — scraper and inspection helpers used to build/validate `states.json`.
- Added: `src/utils/TestUtil.ts` — small helper for attaching screenshots to test info.
- Updated: `README.md` — documents `SMK_SEED`, `RandomUtil`, and smoke selection behavior.

## How to Verify

1. Install deps: `npm install`
2. Run the smoke suite (chromium):

```bash
npx playwright test src/tests/smoke --project=chromium
```

3. Reproduce a deterministic selection (example):

```bash
SMK_SEED=42 npx playwright test src/tests/smoke/home/homeSmoke.spec.ts -g "SMK-020" --project=chromium
```

4. Inspect attached test artifacts in `test-results/` or open the HTML report with:

```bash
npx playwright show-report
```
